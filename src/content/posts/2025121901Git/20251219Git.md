# *一、Github*
## 1.什么是Github？
- Github是一个面向开源及私有软件项目的托管平台，因为只支持Git作为唯一的版本库  格式，故名Github。Github提供的功能包括：
- 核心功能
‌代码托管‌：提供存储库（Repository）来托管代码，支持 Git 版本控制系统，开发者可以追踪代码变更、管理分支和合并代码。‌‌
- ‌协作工具‌：通过 Pull Request 实现代码审查，Issues 用于任务跟踪，Wiki 和 Projects 辅助项目管理，支持全球开发者协作。‌‌
- ‌自动化与扩展‌：内置 GitHub Actions 实现持续集成/持续部署（CI/CD），并可通过 Marketplace 集成第三方工具。‌‌
- 与 Git 的关系
Git 是本地运行的分布式版本控制系统，而 GitHub 是基于 Git 的在线平台，提供 Web 界面和社交化协作功能，简化了代码共享与团队协作流程。‌‌
- 应用场景
‌开源项目‌：全球最大的开源社区，开发者可 Fork 项目、提交贡献并参与协作。‌‌
- ‌企业开发‌：支持私有仓库和权限管理，适用于团队开发与代码安全控制


## 2.如何使用Github？

2.1准备阶段：
   - 访问[Github](https://github.com/)
   - 点击右上角选择`“Sign in”`注册Github账号（可用邮箱注册） 
   
2.2仓库创建：

- 2.2.1、NewRepository-注册完毕且完成登录后点击右上角在下拉菜单
- 2.2.2、Repository name-输入仓库名称（如：`“username.github.io”`）
- 2.2.3、Description-输入仓库描述（可选）
- 2.2.4、Public/Private-选择仓库类型（公开或私有/ 推荐公开）
- 2.2.5、create repository-点击创建仓库
- 2.2.6、uploading an existing file-上传文件（如：`“index.html”`）
- 2.2.7、Push-提交文件（）

2.3部署网站：
  - 2.3.1点击`“Settings”`
  - 2.3.2点击`“Pages”`
  - 2.3.3选择`“Source”`为`“Deploy from a branch”`
  - 2.3.4选择Branchx-选择`“main/root”`
  - 2.3.5点击`“Save”`
  - 2.3.6等待几分钟后刷新网页    
  - 2.3.7回到2.3.1--2.3.2，可以看到域名为 (https://username.github.io/shilicangku/)


**注：1. 此方法只适用于静态网页，如html文件，css文件，js文件等。2.若域名未生效，请检查DNS设置，或稍后再试**

# *二、Github Desktop*
## 1.什么是Github Desktop？
- (●'◡'●)Github Desktop是Github官方推出的跨平台的开源Git客户端，可以让用户高效地管理Github仓库。
- (●'◡'●)支持Windows、Mac和Linux，可以让用户在本地编辑文件，并将修改同步到Github仓库。Github Desktop支持多种操作，包括克隆、创建、删除、推送、拉取、合并分支等。
## 2.如何使用Github Desktop？
- 2.1下载安装：
  - 访问[Github Desktop](https://desktop.github.com/)
  - 点击`“Download for Windows”`下载安装包
  - 运行安装包，选择安装路径，点击`“Next”`
  - 点击`“Install”`完成安装
- 2.2登录Github：
  - 点击`“Sign in”`登录Github账号（可用邮箱注册）
  - 输入账号密码，点击`“Sign in”`
- 2.3克隆仓库：
  - 点击`“Clone a repository from the internet”`
  - 输入仓库地址，点击`“Clone”`
  - 选择本地路径，点击`“Clone”`
- 2.4编辑文件：
  - 点击`“Repository”`
  - 点击`“Repository”`
  - 点击`“Changes”`
  - 点击`“New File”`或`“New Folder”`创建新文件或文件夹
  - 编辑文件内容
  - 点击`“Commit to main”`提交修改
- 2.5推送修改：
  - 点击`“Push origin”`
  - 输入账号密码，点击`“OK”`
  - 等待推送完成
- 2.6查看修改：
  - 点击`“Repository”`
  - 点击`“Repository”`
  - 点击`“History”`
  - 点击`“Refresh”`刷新页面
  - 点击`“Commit”`查看修改内容
- 2.7上传本地文件夹：
  |类别| 步骤 |
  | --- | --- | 
  |已新建仓库| 1.点击左上角`文件`按钮，选择`“添加本地文件”`，按照提示选择本地文件夹即可。|
  |未新建仓库| 1.点击左上角`文件`按钮，选择`“创建新仓库”`，按照提示选择本地文件夹即可。|
